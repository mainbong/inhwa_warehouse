* 명령어 해석기 방식이ㅡ 스크립트 언어이므로 명령 순서대로 실행
* main() 함수를 정의할 필요가 없으며 함수 정의 위치도 중요하지 않음
* 복합 구문에 중괄호 {,} 대신 들여쓰기 사용
* C 언어처럼 명령의 끝을 나타내는 기호가 없음
* 모든 것을 객체로 취급
* 변수는 참조 타입 하나로 통일
* 고급 자료구조를 기본 타입으로 지원
* 함수는 일반화된 정의를 가짐
* 객체지향 및 함수형 패러다임 지원
* 줄 단위 주석은 기존 스크립트 언어와 같이 #을 사용
* 파이썬은 비영리 파이썬 소프트웨어 재단에서 관리하는 개방형, 공동체 기반 개발 모델로 다양한 명령어 해석기 존재
* CPython 은 가장 보편적인 명령어 해석기로 c언어로 구현
* JPython 은 자바 가상 머신용 명령어 해석기
* IronPython은 C#으로 구현된 .NET 플랫폼용 명령어 해석기
* PYPY 는 파이썬으로 작성된 명령어 해석기로 JIT 를 내장해 메모리효율 및 성능이 가장 빠름. 단 최신버전 지원이 느림.
* 여러 줄 주석은 작은따옴표(') 또는 큰 따옴표(") 3개를 시작과 끝 위치에 표시
